## 1. 问题陈述（Problem Statement）

**核心问题**：A 股投资者的决策高度依赖外围市场（美股、日韩）实时动态，但面临三重困境——**信息分散难聚合、影响方向难判断、操作建议难个性化**。现有资讯产品多为"原文搬运+关键词订阅"，既不解释"利好利空及影响"，也无法结合个人持仓与风险特征给出可执行的参考。

**谁会遇到、多频繁**
- 目标用户：A 股个人投资者（散户/大户）、财经自媒体、投顾与研究助理。
- 频次：跟随外围市场交易时间，呈"5×23.5 小时"波浪式高峰——美股盘前/盘后（北京时间凌晨至早盘）、日韩盘中（早盘至午间）为关键窗口，几乎每个交易日多次高价值事件。

**不解决的代价**
- 用户侧：错失外围利好带来的开盘机会，或在利空事件中因信息滞后"踩雷"，直接造成账户亏损。
- 业务侧：东财、同花顺、雪球等竞品已布局"AI 资讯解读+智能选股"，滞后将带来用户流失与差异化定位丧失。

## 2. 目标（Goals）

**用户目标**
- G1：重大外围消息发生后 **≤10 分钟** 内看到结构化解读（五档情绪 + 影响摘要 + 受影响 A 股清单）。
- G2：跨市场信息搜集与判断耗时从数十分钟降至分钟级（降幅 **≥70%**）。
- G3：输入任一股票代码/名称，即可汇集其相关全部消息并按情绪/分类呈现。
- G4：获得"像私人顾问一样"、结合本人持仓与风险特征的个股量价分析与操作参考（持仓/买入/卖出/留底仓卖出）。

**业务目标**
- G5：MVP 上线 30 天内资讯解读功能渗透率（使用用户/活跃用户）**≥25%**。
- G6：个人投顾模块成为差异化留存抓手，使用用户次日留存提升 **≥5pp**。
- G7：为增值服务（深度研报、机构版、个性化预警）积累画像与入口。

## 4. 用户故事（User Stories）

**P0 核心路径**
- US1（散户）：作为 A 股散户，我希望打开网页就能看到按时间倒序的"外围消息 + AI 五档情绪 + 影响摘要 + 受影响 A 股"卡片流，且卡片背景颜色直接告诉我利好利空。
- US2（散户）：作为用户，当一条美股/日韩重大消息出现，我希望收到推送提醒，尤其优先推送我关注的股票池。
- US3（散户）：作为用户，我希望每张卡片标注情绪（利好/利空/中性偏利好/中性/中性偏利空）与一句话影响摘要。
- US4（散户）：作为用户，我希望卡片按"投研报告/企业公告/行业调研"等分类用不同颜色标签区分，便于快速识别消息性质。
- US5（研究/散户）：作为用户，我希望输入股票代码或名称即可检索该股票相关的所有消息。
- US6（散户/投顾）：作为用户，我希望看到"受影响 A 股"列表（代码、名称、关联逻辑）。

**P0 个人投顾**
- US7（大户）：作为持仓用户，我希望系统像私人顾问一样，对我持有的每只股票做量价技术分析并给出"持仓/买入/卖出/留底仓卖出"建议及原因。
- US8（所有用户）：作为用户，我希望投顾建议结合我的资金体量、仓位、操作习惯、历史操作与学习能力，而非千篇一律。

**P1 增强**
- US9（自媒体）：作为财经自媒体，我希望一键复制"消息+解读+标的"结构化内容。
- US10（高频用户）：作为用户，我希望按板块/概念/自选股筛选受影响标的。

**边界 / 异常**
- US11（空状态）：无匹配消息时展示引导式空状态。
- US12（低置信度）：AI 对情绪判定置信度低时，明确提示"判断不确定，仅供参考"。
- US13（采集失败）：单数据源不可用时不展示残缺解读，自动降级。

## 5. 需求（Requirements）

> MoSCoW：P0=Must，P1=Should/Could，P2=Won't(this time，架构预留)。

### 5.1 跨市场实时采集与分析（对应诉求①）

**R1 多市场实时资讯采集（P0）**
- 描述：实时采集 A 股、美股、日韩股市相关财经新闻与公告。
- 默认数据源：A 股（财经媒体 RSS/API、交易所/巨潮公告）；美股（权威媒体、SEC、公司 IR）；日韩（日经、韩联社等，注意语言与合规）。
- 接受标准：
  - [ ] 新消息 ≤1 分钟进入处理流水线；核心源轮询 ≤1 分钟。
  - [ ] 单源故障隔离，不影响其他源。
  - [ ] 数据源增删可配置化，无需发版。

**R2 内容清洗、去重与结构化（P0）**
- 描述：正文抽取、语种识别、相似度去重、实体识别（公司/行业/概念/宏观）。
- 接受标准：
  - [ ] 同一事件多篇报道聚合为一条，不重复推送。
  - [ ] 提取字段：标题、正文、来源、发布时间、涉及实体、原市场、原文链接。
  - [ ] 广告/软文/非财经噪声被过滤或降权。

**R3 五档情绪判定与影响分析（P0，对应诉求①④）**
- 描述：基于 LLM + 金融知识，对每条事件判定**五档方向**并给出置信度与影响摘要。
- 五档定义：
  - **利好**：明确积极，预期推动相关 A 股上行。
  - **利空**：明确消极，预期压制相关 A 股。
  - **中性偏利好**：整体中性但含积极因素，略偏正向。
  - **中性**：无明显方向性影响。
  - **中性偏利空**：整体中性但含消极因素，略偏负向。
- 接受标准（Given/When/Then）：
  - Given 一条已结构化事件
  - When 分析流水线处理
  - Then 输出方向（五档之一）、置信度(0-1)、一句话影响摘要、影响维度（情绪面/基本面/资金面）。
  - Then 置信度 < 阈值（默认 0.6）时前端展示"判断不确定，仅供参考"。
  - Then 判定需可解释、禁止编造；无法判断宁可标"中性/不确定"。

**R4 受影响 A 股映射（P0，对应诉求①）**
- 描述：将事件映射到"受影响 A 股"清单并说明关联逻辑。
- 映射依据：产业链上下游 / 同业竞争替代 / 供应商客户依赖 / 概念板块标签。
- 接受标准：
  - [ ] 输出含：代码、名称、所属板块、关联类型、一句话关联理由。
  - [ ] 每支标的至少一条可追溯关联逻辑，不得无理由罗列。
  - [ ] 核心产业链事件命中 ≥80% 相关 A 股（评测集衡量）。
  - [ ] 无映射时明确标注"暂无直接 A 股影响标的"。

### 5.2 消息分类与情绪配色（对应诉求③④）

**R5 消息分类体系与标签配色（P0，对应诉求③）**
- 描述：对每条消息标注**分类**，并以不同颜色标签（badge）区分。
- 默认分类与配色（标签/角标色，独立于情绪背景色）：

| 分类 | 含义 | 标签色（默认假设 HEX） |
|---|---|---|
| 企业公告 | 上市公司正式公告/财报 | 靛蓝 `#3B5BDB` |
| 投研报告 | 券商/机构研究报告 | 紫 `#7048E8` |
| 行业调研 | 行业走访/调研纪要 | 橙 `#E8590C` |
| 财经快讯 | 即时新闻快报 | 灰 `#495057` |
| 政策动态 | 监管/产业政策 | 赭金 `#B8860B` |
| 宏观数据 | GDP/CPI/就业等 | 青绿 `#0CA678` |
| 海外市场 | 美股/日韩原发 | 深蓝 `#1864AB` |

- 接受标准：
  - [ ] 每条消息展示分类标签且颜色可区分。
  - [ ] 分类可配置扩展（新增类别不改前端结构）。
  - [ ] 分类与情绪为两个独立维度（标签色 ≠ 背景色）。

**R6 情绪背景配色规范（P0，对应诉求④）**
- 描述：消息卡片**背景色**按 AI 情绪判定着色，直观区分方向。
- 配色规范（卡片背景淡色，默认假设 HEX）：

| 情绪 | 背景色 | HEX | 文字/强调色 |
|---|---|---|---|
| 利好 | 淡红 | `#FFF0F0` | `#C92A2A` |
| 利空 | 淡绿 | `#F0FFF4` | `#2B8A3E` |
| 中性 | 蓝 | `#F0F7FF` | `#1971C2` |
| 中性偏利好 | 粉 | `#FFF0F5` | `#C2255C` |
| 中性偏利空 | 青 | `#F0FFFF` | `#0C8599` |

- 接受标准：
  - [ ] 卡片背景严格按上表映射，与分类标签色互不冲突。
  - [ ] 提供高对比度模式（ accessibility ）与色盲友好辅助（图标/文字双重标识）。
  - [ ] 配色集中管理（设计 token），支持主题切换。

### 5.3 检索（对应诉求②）

**R7 按代码/名称检索消息（P0，对应诉求②）**
- 描述：用户输入股票代码或名称，汇集该标的相关全部消息。
- 接受标准：
  - [ ] 支持代码（如 `300308`）、简称（如 `中际旭创`）、全称检索。
  - [ ] 结果按时间倒序，复用 R6 情绪背景 + R5 分类标签。
  - [ ] 支持"仅看利好/利空"等情绪筛选。
  - [ ] 检索命中关联映射（即被 R4 列为受影响标的的消息也应纳入）。

### 5.4 个人投顾（对应诉求⑤）

**R8 用户画像模型（P0，对应诉求⑤）**
- 描述：建立并维护用户个人画像，支撑个性化投顾。
- 画像字段：
  - 持仓股票（代码/名称/成本/数量）
  - 资金体量（总资金规模区间）
  - 当前仓位（总仓位占比）
  - 操作习惯（短线/中线/长线、交易频率）
  - 历史操作（买卖记录、胜率、止盈止损风格）
  - 个人偏好（风险偏好、行业偏好）
  - 个人学习能力（新手/进阶/资深——影响解读详略）
- 接受标准：
  - [ ] 用户可手动维护画像，并授权从券商/自选股导入（合规前提下）。
  - [ ] 画像数据加密存储，明确告知用途与授权范围。

**R9 量价技术分析与操作建议（P0，对应诉求⑤）**
- 描述：对个人持仓股票，结合**量价关系**做技术分析，输出四类建议之一并解释原因。
- 建议类型：**持仓 / 买入 / 卖出 / 留底仓卖出**。
  - 持仓：继续持有，理由（如量价配合健康、趋势未破）。
  - 买入：加仓/建仓，理由（如缩量回调企稳、量价齐升启动）。
  - 卖出：清仓，理由（如高位放量滞涨、量价顶背离）。
  - 留底仓卖出：减仓兑现利润但保留底仓，理由（如短线超买但中线逻辑未坏）。
- 分析维度（重点量价）：
  - 量价配合（放量上涨/缩量整理/放量滞涨/地量见底）
  - 趋势与均线、关键支撑压力
  - 结合用户仓位/资金体量给出"建议操作仓位占比"
- 接受标准（Given/When/Then）：
  - Given 用户持有股票 X 且画像完整
  - When 用户查看"我的投顾"
  - Then 对 X 输出：量价结论 + 建议类型 + 原因 + 结合本人仓位/习惯的备注。
  - Then 每条建议附"AI 生成、仅供参考、不构成投资建议"声明（见 R12）。
  - Then 当数据不足时提示"暂无法给出建议"，不强行输出。
- 约束：建议须可解释、基于公开技术面，禁止承诺收益。

### 5.5 推送优先级（对应诉求⑦）

**R10 优先推送股票池（P0，对应诉求⑦）**
- 描述：对指定股票池消息优先推送；叠加用户持仓。
- 默认优先池（硬编码可配置）：
  - 英伟达(NVDA)、三星(005930.KS)、海力士(000660.KS)、AMD、美光(MU)、英特尔(INTC)、康宁(GLW)
  - 中际旭创(300308)、东山精密(002384)、寒武纪(688256)、兆易创新(603986)、长江存储、长鑫存储
  - + 用户持仓股票（动态）
- 接受标准：
  - [ ] 优先池消息在采集后优先进入推送队列、降低时延。
  - [ ] 优先池可后台配置与用户自定义叠加。
  - [ ] 与 R7 检索、R4 映射联动（优先池标的被影响时也触发）。

### 5.6 视觉与交互（对应诉求⑥）

**R11 水墨炫酷网页 UI（P0，对应诉求⑥）**
- 描述：网页端整体白色背景，动画采用水墨风格，整体炫酷现代。
- 设计规范（默认假设）：
  - 背景：纯白 `#FFFFFF` / 微暖白 `#FAFAFA`。
  - 水墨元素：卡片入场用"墨晕扩散"SVG/CSS 动画；分割线用"笔锋"笔触；加载态用"墨滴晕开"；主题切换/刷新用水墨转场。
  - 炫酷感：玻璃拟态卡片（半透明+模糊）、轻渐变、流畅 60fps 过渡、悬浮微交互。
  - 字体：中文优先思源/系统无衬线，标题可配书法感字体点缀。
- 接受标准：
  - [ ] 满足白底 + 水墨动画 + 现代炫酷三要素。
  - [ ] 动画可关闭（性能/无障碍），弱网下降级为静态。
  - [ ] 情绪背景色（R6）与水墨风格协调共存。

### 5.7 质量与风控（跨 P0/P1）

**R12 合规与免责（P0，强约束）**
- 描述：所有 AI 解读/投顾建议附"AI 生成、仅供参考、不构成投资建议"声明；内容符合财经信息合规。
- 接受标准：每卡片/推送/投顾页均含声明；敏感内容走审核或屏蔽；R9 建议不得含收益承诺。

**R13 AI 质量评测（P0）**
- 描述：建立标注评测集，监控五档判定准确率、分类准确率、映射召回率。
- 接受标准：MVP 前五档判定准确率（人工评测）≥80%；上线后周维度监控。

**R14 个性化与板块聚合（P1）**
- 描述：按自选股/板块/概念订阅；板块级聚合视图（今日外围利空/利好最集中板块）。

**R15 历史回溯与多市场扩展（P2）**
- 描述：相似事件历史表现；港股/欧股接入；机构 API。

## 6. 成功指标（Success Metrics）

**领先指标**
| 指标 | 定义 | 成功阈值 | 评估时点 |
|---|---|---|---|
| 功能渗透率 | 使用解读功能用户 / 活跃用户 | ≥25% @30天 | 30天 |
| 端到端时延 | 消息发布→可见解读 | 核心 ≤10 分钟 | 持续 |
| 五档判定准确率 | 人工抽检方向正确率 | ≥80% | 7/30天 |
| 分类准确率 | 分类标签人工抽检 | ≥90% | 30天 |
| 检索使用率 | 使用代码/名称检索的用户占比 | ≥15% | 30天 |
| 投顾模块使用率 | 查看"我的投顾"用户占比 | ≥20% | 30天 |
| 推送打开率 | 推送点击 / 送达 | ≥15% | 14天 |
| 误读/投诉率 | 举报误读次数 / 推送量 | ≤1% | 持续 |

**滞后指标**
- 次日留存：使用投顾/解读用户次日留存较未使用提升 ≥5pp。
- 满意度：资讯/投顾模块评分 ≥4.2/5。
- 增值转化：为付费功能贡献入口点击基线。
