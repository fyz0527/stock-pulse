## 1. 问题陈述（Problem Statement）

**核心问题**：A 股投资者的决策高度依赖外围市场（美股、日韩股市）的实时动态，但跨市场信息高度分散、专业解读门槛高、且传递不及时，导致投资者在重大外部事件发生时"看不懂、找不到、来不及"。

**谁会遇到、多频繁**：
- 目标用户：A 股个人投资者（散户/大户）、财经自媒体、投顾与研究助理。
- 频次：跟随外围市场交易时间，呈现"5×23.5 小时"的波浪式高峰——美股盘前/盘后（北京时间凌晨至早盘）、日韩盘中（北京时间早盘至午间）为关键窗口，几乎每个交易日都有多次高价值事件。

**不解决的代价**：
- 用户侧：错失外围利好带来的开盘机会，或在利空事件中因信息滞后"踩雷"，直接造成账户亏损。
- 业务侧：东财、同花顺、雪球等竞品已在布局"AI 资讯解读 + 智能选股"能力，若我们滞后，将面临用户流失与差异化定位丧失的竞争性风险。

**问题证据（默认假设，待补充真实数据）**：
- 内部调研/竞品观察显示，用户对"一条美股消息到底利好利空 A 股哪只票"的诉求强烈。
- 现有资讯产品多为"原文搬运 + 关键词订阅"，缺乏对"影响传导链条"的结构化解读。

## 2. 目标（Goals）

**用户目标（用户得到什么）**
- G1：重大外围消息发生后，用户在 **N 分钟内（默认目标：美股/日韩盘前盘后核心消息 ≤ 10 分钟）** 看到结构化解读（利好/利空 + 影响摘要 + 受影响 A 股清单）。
- G2：将"跨市场信息搜集与判断"的耗时从当前的 **数十分钟级降至分钟级（默认假设降幅 ≥ 70%）**。
- G3：用户能清晰理解"这条消息 → 影响哪个板块/哪只 A 股 → 为什么"。

**业务目标（公司得到什么）**
- G4：资讯解读功能成为日活（DAU）与次日留存（Next-Day Retention）的新抓手，目标 MVP 上线 30 天内功能渗透率（使用用户/活跃用户）≥ 25%。
- G5：为后续增值服务（深度研报、个性化预警、机构版）积累数据与入口。

**衡量方式**：以上目标中时间类指标由服务端埋点 + 消息链路时间戳计算；渗透/留存指标由数据平台看板评估（详见第 6 节）。

## 3. 非目标（Non-Goals）

| 编号 | 非目标 | 为什么不在 v1 |
|---|---|---|
| NG1 | 提供个股买卖建议 / 自动交易下单 | 合规风险高、超出"信息解读"定位，属独立业务线 |
| NG2 | 覆盖期货、外汇、加密货币、债券实时聚合 | 范围过大，v1 聚焦权益市场三大核心盘面 |
| NG3 | 港股 / 欧股的实时聚合 | 后续 P2 扩展，v1 先验证"跨市场→A 股传导"主链路 |
| NG4 | 历史回测 / 策略回测工具 | 属量化研究范畴，非当前用户核心痛点 |
| NG5 | 社区 / UGC / 评论互动 | 避免内容合规与运营成本，v1 以"权威资讯 + AI 解读"为主 |
| NG6 | 直接对接交易所行情交易接口 | v1 仅做资讯与标的映射，行情展示可复用现有行情能力 |

## 4. 用户故事（User Stories）

**优先级 P0（核心路径）**

- US1（散户）：作为 A 股散户，我希望打开 App 就能看到按时间倒序的"外围消息 + AI 解读 + 受影响 A 股"卡片流，这样我不用自己拼凑信息。
- US2（散户）：作为 A 股散户，当一条美股/日韩重大消息出现时，我希望收到推送提醒，这样我不会错过盘前决策窗口。
- US3（散户）：作为 A 股散户，我希望每张卡片明确标注"利好 / 利空 / 中性"以及一句话影响摘要，这样我能快速判断情绪方向。
- US4（散户/投顾）：作为用户，我希望在卡片中看到"受影响的 A 股股票"列表（含代码、名称、关联逻辑），这样我能直接定位关注标的。
- US5（研究助理）：作为研究助理，我希望能看到消息的"影响传导逻辑"（如：英伟达业绩 → 算力产业链 → 国产 GPU 概念），这样我能用于进一步研究。

**优先级 P1（增强体验）**

- US6（大户）：作为高频用户，我希望按"板块 / 概念 / 自选股"维度筛选受影响标的，这样我能只看和我持仓相关的内容。
- US7（自媒体）：作为财经自媒体，我希望一键复制"消息 + 解读 + 标的"的结构化内容，这样我能快速产出短内容。
- US8（所有用户）：作为用户，我希望看到 AI 对该消息"影响强度"的评分（高/中/低），这样我能区分噪音与关键事件。

**边界 / 异常故事**

- US9（空状态）：作为新用户，当暂无匹配我订阅条件的消息时，我希望看到引导式空状态而非空白页。
- US10（低置信度）：当 AI 对一条消息的利好利空判定置信度较低时，我希望被明确告知"判断不确定，仅供参考"，而非武断给出结论。
- US11（采集失败）：当某数据源（如某美股财经媒体）临时不可达时，我希望系统自动降级并不展示残缺解读，而不是卡住整个信息流。

## 5. 需求（Requirements）

> 分类采用 MoSCoW（Must / Should / Could / Won't）。下文 P0=Must、P1=Should/Could、P2=Won't(this time，但需架构预留)。

### 5.1 数据采集与接入（P0）

**R1 多市场实时资讯采集**
- 描述：系统需实时采集 A 股、美股、日韩股市相关的财经新闻与公告。
- 数据源（默认假设）：
  - A 股：主流财经媒体 RSS/API、交易所公告、巨潮/东方财富等。
  - 美股：权威财经媒体、SEC 公告、公司 IR 发布。
  - 日韩：日经、韩联社等可获取源（需注意语言与合规）。
- 采集频率：核心源轮询 ≤ 1 分钟；事件型源（如公告）近实时。
- 接受标准：
  - [ ] 给定一条新发布的样本消息，系统在 ≤ 1 分钟内进入处理流水线。
  - [ ] 单源故障不影响其他源采集（隔离）。
  - [ ] 支持数据源增删配置化，无需发版。

**R2 内容清洗、去重与结构化**
- 描述：对原始资讯做正文抽取、语种识别、相似度去重、实体识别（公司/行业/概念/宏观事件）。
- 接受标准：
  - [ ] 同一事件的多篇报道聚合为一条"事件"，不重复推送。
  - [ ] 提取结构化字段：标题、正文、来源、发布时间、涉及实体、原始市场、原文链接。
  - [ ] 明显噪声（广告、软文、非财经）被过滤或降权。

### 5.2 AI 影响分析（P0 — 产品核心价值）

**R3 利好 / 利空 / 中性判定**
- 描述：基于 LLM + 金融知识，对每条事件判定方向（利好 / 利空 / 中性）并给出置信度。
- 接受标准（Given/When/Then）：
  - Given 一条已结构化的财经事件
  - When 分析流水线处理该事件
  - Then 输出方向标签（利好/利空/中性）、置信度（0-1）、一句话影响摘要。
  - Then 当置信度 < 阈值（默认 0.6）时，前端展示"判断不确定，仅供参考"。
- 约束：判定需可解释（附带关键依据），禁止凭空编造；对无法判断的事件宁可标"中性/不确定"。

**R4 影响传导分析与受影响 A 股映射**
- 描述：系统需将事件映射到"受影响的 A 股股票"清单，并说明关联逻辑。
- 映射依据（默认假设，需知识库支撑）：
  - 产业链上下游（如：英伟达 → 算力/CPO/PCB → 相关 A 股）。
  - 同业竞争 / 替代（如：某日企受挫 → 国产替代受益标的）。
  - 供应商 / 客户依赖（如：苹果链、特斯拉链 A 股供应商）。
  - 概念板块标签（半导体、新能源、医药等）。
- 接受标准：
  - [ ] 输出标的列表含：股票代码、名称、所属板块、关联类型、一句话关联理由。
  - [ ] 每支标的至少附一条可追溯的关联逻辑，不得无理由罗列。
  - [ ] 映射覆盖度目标：核心产业链事件命中 ≥ 80% 相关 A 股（由评测集衡量）。
  - [ ] 对"无明显 A 股映射"的事件，明确标注"暂无直接 A 股影响标的"。

**R5 影响强度评分（P1）**
- 描述：对事件给出影响强度（高/中/低）及可能影响维度（情绪面/基本面/资金面）。
- 接受标准：强度与"标的数量、市场关注度、历史相似事件反应"挂钩，可解释。

### 5.3 展示与触达（P0）

**R6 资讯解读卡片流（前端）**
- 描述：移动端/Web 端展示倒序卡片流，单卡包含：市场标签、时间、标题、方向标签、摘要、受影响 A 股列表（可展开）、原文链接。
- 接受标准：
  - [ ] 卡片按发布时间倒序，利好（红）/利空（绿）按 A 股习惯配色（涨红跌绿）。
  - [ ] 受影响 A 股点击可跳转行情/详情。
  - [ ] 支持按市场（A股/美股/日韩）筛选。

**R7 推送与提醒（P0）**
- 描述：对高影响强度或用户订阅相关的事件推送提醒。
- 接受标准：
  - [ ] 用户可配置"仅推送高影响 / 关注板块"等开关。
  - [ ] 推送频次有防打扰策略（如聚合窗口、单用户上限）。
  - [ ] 重大利空/利好事件推送延迟 ≤ 10 分钟。

### 5.4 个性化与扩展（P1 / P2）

**R8 个性化订阅与筛选（P1）**
- 描述：用户可按自选股、板块、概念订阅，仅看相关解读。
- 接受标准：配置生效实时；与现有自选股体系打通。

**R9 板块级聚合视图（P1）**
- 描述：将受影响标的按板块聚合，展示"今日外围利空/利好最集中的 A 股板块"。

**R10 历史回溯与相似事件（P2）**
- 描述：展示"历史上类似事件发生后相关 A 股表现"，辅助判断（非预测）。

**R11 多市场扩展与开放（P2）**
- 描述：架构预留港股、欧股接入；预留面向机构的 API/数据导出。

### 5.5 质量与风控（跨 P0/P1）

**R12 AI 解读质量评测**
- 描述：建立标注评测集，定期评估方向判定准确率、映射召回率。
- 接受标准：MVP 上线前方向判定准确率（人工评测）≥ 80%；上线后周维度监控。

**R13 合规与免责**
- 描述：所有 AI 解读需附"AI 生成、仅供参考、不构成投资建议"免责声明；内容需符合财经信息合规要求。
- 接受标准：每卡片/推送均含免责标识；敏感内容走合规审核或屏蔽。

## 6. 成功指标（Success Metrics）

**领先指标（上线后数天至数周）**
| 指标 | 定义 | 成功阈值 | 延伸目标 | 评估时点 |
|---|---|---|---|---|
| 功能渗透率 | 使用解读功能的用户 / 活跃用户 | ≥ 25% @30天 | ≥ 40% | 上线 30 天 |
| 激活率 | 看到 ≥ 1 张解读卡片并展开标的的用户占比 | ≥ 60% | — | 上线 7 天 |
| 端到端时延 | 消息发布 → 用户可见解读 | 核心消息 ≤ 10 分钟 | ≤ 5 分钟 | 持续监控 |
| 推送打开率 | 推送点击 / 推送送达 | ≥ 15% | ≥ 25% | 上线 14 天 |
| 解读准确率 | 人工抽检方向判定正确率 | ≥ 80% | ≥ 90% | 上线 7/30 天 |
| 误推/投诉率 | 用户举报"误读/无关"次数 / 推送量 | ≤ 1% | ≤ 0.5% | 持续监控 |

**滞后指标（数周至数月）**
- 次日留存：使用解读功能的用户次日留存较未使用用户提升 ≥ 5pp。
- 用户满意度（NPS/评分）：资讯解读模块评分 ≥ 4.2/5。
- 增值转化：为后续付费功能贡献入口点击与转化基线。

**测量方式**：服务端埋点（事件时间戳、曝光/点击）+ 数据平台看板 + 人工抽检评测集。

## 8. 时间线考虑（Timeline Considerations）

**硬约束（默认假设，待确认）**
- 无明确对外承诺日期；建议对标下一次版本大版本或财报季窗口。

**依赖**
- 依赖数据源商务/技术接入（Q1）、产业链知识库建设（Q2）、合规方案确认（Q4）。
- 依赖现有行情/自选股能力复用（R6/R8）。

**建议分阶段**
- **阶段一（MVP / P0）**：R1–R7 + R12 + R13。验证"采集→判定→映射→展示→推送"主链路。
- **阶段二（P1 增强）**：R5 强度评分、R8 个性化、R9 板块聚合、R7 推送精细化。
- **阶段三（P2 扩展）**：R10 历史回溯、R11 多市场/API。

**里程碑建议（相对排期，默认假设）**
- M1 数据源接入与清洗可用：第 2–3 周
- M2 AI 判定+映射评测达标（≥80%）：第 5–6 周
- M3 MVP 端到端可用（含前端+推送）：第 8 周
- M4 灰度 + 全量：第 9–10 周

## 附录 C：术语表

- **利好/利空**：对 A 股相关标的股价的积极/消极预期影响。
- **受影响 A 股**：因该事件在产业链、同业、供需或概念层面产生关联的中国 A 股上市公司。
- **影响强度**：AI 对事件冲击程度的评分（高/中/低）。
