# 深化设计（二）：推送频控方案
## 对应 PRD R10 / R7 — 优先池优先、但不打扰用户

> 目标：对指定优先股票池（NVDA/三星/海力士/AMD/美光/英特尔/康宁/中际旭创/东山精密/寒武纪/兆易创新/长江存储/长鑫存储 + 用户持仓）做到**低时延优先推送**，同时通过分层、聚合、限流三道防线避免通知疲劳与卸载。

---

## 1. 设计原则

1. **重要且不烦**：高价值（优先池 + 强情绪 + 持仓）即时达；低价值进摘要。
2. **聚合优于轰炸**：同一标的短时间多事件合并为一条。
3. **用户主权**：频控参数可配置，用户可一键切换"实时/每15分/每日摘要"。
4. **静默有度**：夜间免打扰，但极端重要事件可"强提醒"覆盖（带上限）。
5. **可观测**：打开率、退订率、举报率实时看板，超阈值自动降温。

---

## 2. 通道矩阵

| 通道 | 时效 | 适用层级 | 备注 |
|---|---|---|---|
| 站内（App 内消息中心） | 实时 | 全部 | 永不静默，作为兜底记录 |
| 系统推送（Push） | 近实时 | P0/P1 | 受 OS 静默时段限制 |
| 短信 | 分钟级 | 仅 P0 极端重要 | 默认关闭，需用户开启；成本最高 |

---

## 3. 优先级分层

每条待推事件先打 `push_tier`：

| 层级 | 触发条件 | 处理 |
|---|---|---|
| **P0 即时** | 命中优先池(R10) 或 用户持仓，且 情绪∈{利好,利空} 或 impact_level=高 | 进即时队列，打破聚合延迟 |
| **P1 聚合** | 优先池/持仓但情绪中性偏× 或 非池内但 impact=高 | 进入 15 分钟聚合窗口后推送 |
| **P2 摘要** | 其余（中性、低影响、非池内） | 不单独推，进"每日/每半日摘要" |

> 优先池与持仓为"准入加速"，情绪/影响强度决定层级。两者叠加即为最高优先。

---

## 4. 聚合 / 合并窗口

- **按标的聚合**：同一股票（代码）在 `AGG_WINDOW=15min` 内的多条事件合并为一条：
  `「中际旭创」相关 3 条外围消息：2 利好 / 1 中性偏利空`
- **按主题聚合**：宏观类（如美联储）多条合并为"美联储动态汇总"。
- P0 即时事件豁免聚合（但同一标的 `COOLDOWN=30min` 内不重复即时推，除非情绪方向翻转）。

---

## 5. 限流规则

### 5.1 用户级（per-user）
| 参数 | 默认 | 说明 |
|---|---|---|
| `MAX_PER_HOUR` | 8 | 单用户每小时系统推送上限 |
| `MAX_PER_DAY` | 30 | 单用户每日上限 |
| 超限处理 | 余量转入下一周期或降为站内 |

### 5.2 标的级（per-stock）
| 参数 | 默认 | 说明 |
|---|---|---|
| `STOCK_COOLDOWN` | 30 min | 同标的两次即时推最小间隔 |
| 翻转豁免 | 开启 | 利空→利好方向反转可立即再推 |

### 5.3 全局（global，保护通道与成本）
| 参数 | 默认 | 说明 |
|---|---|---|
| `GLOBAL_RATE` | 按容量压测 | 全站每秒推送吞吐上限，令牌桶 |
| 背压 | — | 超限时 P1/P2 排队，P0 优先 |

---

## 6. 免打扰与强提醒覆盖

- **静默时段**：`QUIET_START=22:00` – `QUIET_END=07:30`（用户时区，可改）。
  - 静默内：系统推送挂起，仅写入站内；P0 极端重要可"强提醒"覆盖。
- **强提醒（Override）**：仅限 P0 且命中"用户持仓重大利空/利好"或"优先池 + impact=高"。
  - 强提醒每日每用户上限 `MAX_OVERRIDE=2`，防滥用。
  - 用户可在设置关闭强提醒。

---

## 7. 用户可控项（设置页）

- 推送总开关
- 频率模式：实时 / 每15分钟 / 每日摘要
- 仅优先池 + 持仓
- 按情绪过滤（如"只看利空"）
- 按分类过滤（如"只要企业公告/投研报告"）
- 强提醒开关
- 静默时段自定义

---

## 8. 决策流程图

```
事件到达
  │
  ├─ 是否命中 优先池/持仓？ ──否──► 非P0
  │        │是
  │        ▼
  ├─ 情绪/影响层级？
  │    强(利好/利空/高影响) ──► P0 即时
  │    中(中性偏×) ──────────► P1 聚合(15min)
  │    弱/中性 ──────────────► P2 摘要
  │
  ▼ 进入对应队列
[P0] 检查 STOCK_COOLDOWN / 翻转豁免
     │ 通过 → 检查用户级/全局限流 → 系统推送(静默时段? 强提醒 or 站内)
[P1] 聚合窗口 → 合并 → 限流 → 系统推送
[P2] 累计 → 摘要周期 → 站内/推送
  │
  ▼ 全部写入站内消息中心（兜底）
```

---

## 9. 配置参数总表（可在后台热更新）

| 参数 | 默认值 | 说明 |
|---|---|---|
| AGG_WINDOW | 15 min | 同标的聚合窗口 |
| STOCK_COOLDOWN | 30 min | 同标的即时推冷却 |
| MAX_PER_HOUR | 8 | 用户每小时上限 |
| MAX_PER_DAY | 30 | 用户每日上限 |
| MAX_OVERRIDE | 2 / 日 | 强提醒上限 |
| QUIET_START / END | 22:00 / 07:30 | 静默时段 |
| P0_IMPACT_MIN | 高 | P0 情绪/影响阈值 |

---

## 10. 监控指标与护栏

| 指标 | 预警线 | 动作 |
|---|---|---|
| 推送打开率 | < 12% | 检查内容质量/频次 |
| 退订率(日) | > 1.5% | 自动下调默认频次 |
| 举报率 | > 1% | 人工复核 + 降权源 |
| P0 端到端时延 | > 10 min | 告警链路 |
| 强提醒触达率 | — | 保障极端事件不漏 |

> 护栏逻辑：当退订率连续 3 日超阈值，自动将新用户默认频率从"实时"降为"每15分钟"，并弹窗引导自定义。
