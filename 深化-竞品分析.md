# 深化设计（三）：竞品分析
## 验证"跨市场情绪传导 + 个人化投顾 + 水墨差异化"的差异化空间

> 方法：基于公开资料与产品体验整理（核实于 2026-07；上线前建议复审）。对照本产品 7 项核心诉求，评估主要竞品覆盖度，定位差异化 wedge。

---

## 1. 竞品画像

### 1.1 东方财富「妙想」大模型
- 定位：国内首批通过网信办备案的金融垂类大模型（2025-03 全量开放），集成于东方财富 App / Web。
- 能力：信息检索、数据分析、智能研报生成、会议纪要、研报总结、"最强板块"识别。优先用 EDB 数据库/公司公告等权威信源。
- 短板：跨市场"美/日韩→A 股传导"并非独立产品主线；UI 为传统财经风格。

### 1.2 同花顺「问财」
- 定位：自然语言智能选股/量化/技术分析平台，覆盖 A/美/港/宏观/资讯。
- 能力：自然语言交互（"算力板块潜力股"），基于 3000+ 财务指标与实时舆情生成筛选；2025 新增类 DeepSeek"深度思考"模块。2025 营收 60.29 亿（+44%）、净利 32.05 亿（+76%）。
- 短板：偏"选股工具"，非"跨市场事件→情绪→个人投顾"连贯体验；个性化投顾结合用户画像/仓位较弱。

### 1.3 财联社「电报 / 电报解读」
- 定位：7×24 极速财经通讯社，以"电报"为特色，覆盖 A 股与全球资本市场。
- 能力：**「电报解读」将单个事件延展到相关行业与标的**（最接近本产品 R4 受影响 A 股映射）；机构级前瞻视角，曾交出亮眼覆盖战绩。
- 短板：偏资讯/事件侧，缺"私人投顾式量价分析 + 个人画像"，且非跨市场整合产品。

### 1.4 雪球
- 定位：投资社区 + 组合 + 行情资讯。
- 能力：UGC 深度、组合跟踪、个股讨论氛围强；有基础智能资讯。
- 短板：AI 结构化解读与跨市场情绪传导非强项；依赖社区内容质量。

### 1.5 腾讯自选股 / 新浪财经
- 定位：行情 + 资讯 + 自选股提醒。
- 能力：推送提醒、基础分类资讯。
- 短板：AI 解读浅、无个人投顾、UI 常规。

### 1.6 韭研公社 / 通达信
- 韭研公社：研究社区（人驱），深度但非实时 AI。
- 通达信：交易终端 + 强技术分析指标，但 AI/跨市场资讯弱。

### 1.7 国际参考（成熟度标杆）
- **Seeking Alpha**：众包分析 + AI 评级/摘要，事件→个股分析成熟。
- **Benzinga**：快讯 + 信号 +  alerts，推送体系成熟。
- **Bloomberg Terminal**：专业终端， alert 与跨市场数据极强（但面向机构、价高）。

## 3. 差异化定位（Wedge）

**一句话定位**：*首个把"跨市场情绪传导 → 五档可视化 → 私人投顾式量价建议 → 水墨差异化体验 → 指定股票池优先推送"串成一体验的 AI 炒股助手。*

三层级差异化：
1. **传导深度**（别人弱）：A 股投资者最痛的"美股/日韩消息到底利好利空哪只 A 股"，我们做成核心主线（R1–R4 + 映射算法文档）。
2. **个人化深度**（别人无）：结合持仓/仓位/习惯/历史的"量价投顾"（R8–R9），是留存护城河。
3. **体验辨识度**（别人无）：水墨炫酷白底 UI（R11），低成本高记忆点。

## 4. SWOT

| | 内容 |
|---|---|
| **S 优势** | 跨市场传导主线清晰；五档情绪+背景色可视化独创；个人投顾结合画像深度；水墨 UI 辨识度高；优先池可配置 |
| **W 劣势** | 入场晚、无海量用户与数据飞轮；大模型依赖第三方、成本/时延承压；投顾功能合规风险最高 |
| **O 机会** | AI-native 新产品可在 UX 上弯道超车；"美/日韩→A"整合体验尚无人占据；散户对"看得懂的影响"需求强 |
| **T 威胁** | 东财/同花顺有数据+分发+牌照，可快速复制单点功能；监管对 AI 投资建议趋严；数据源合规与成本 |

---

## 5. 战略建议

1. **先打最难的传导主线（R1–R4 + R10）**：这是 incumbent 最难"快速拼成一体验"的部分，且直接命中用户痛点，应作为 MVP 尖刀。
2. **水墨 UI 做品牌锚点**：低成本、高辨识，避免陷入与传统财经 App 的同质视觉竞争。
3. **个人投顾（R9）分批释放**：它是留存护城河，也是最大合规风险——上线即强免责，定位"分析参考"而非"买卖指令"；待合规路径清晰再加深。
4. **以"速度 + 优先池可配置"对标财联社**：在推送时效与个性化优先池上做文章，抢"盘前决策"场景。
5. **不要正面拼数据规模**：不与东财/同花顺比全量数据，而比"解读质量 + 个人化 + 体验"。

## 6. 风险与缺口

| 风险 | 说明 | 缓解 |
|---|---|---|
| 功能被快速复制 | incumbent 可抄单点 | 用"主线整合 + 个人画像数据闭环"建壁垒 |
| 合规压制投顾 | R9 可能受限 | 强免责 + 定位参考 + 合规前置(Q4) |
| 数据源/成本 | 跨市场实时 + LLM 成本高 | 频控(R10文档) + 缓存 + 分层模型 |
| 冷启动无数据 | 画像/映射无反馈 | 冷启动策略（映射算法文档 §10）+ 引导式画像采集 |

> 注：对比结论基于 2026-07 公开资料，竞品迭代快，正式立项前需做一次实地体验复核与功能打分。
